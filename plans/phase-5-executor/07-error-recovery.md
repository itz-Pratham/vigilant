# Phase 5 — Error Recovery

**File:** `src/executor/error-recovery.ts`

## Objective

Define what happens at each failure point and how the executor picks up from incomplete runs. Every executor failure sets `stage = 'blocked'` with a human-readable `blockerReason` — the human decides whether to retry or close.

---

## Failure Scenarios

| Step | Failure cause | Blocker reason written | Recovery on restart |
|---|---|---|---|
| `resolveBaseSha` | GitHub 404 / 403 | "Repository not found or access revoked" | Retry from `not_started` |
| `createBranch` | GitHub 422 (already exists) | None — treated as success | Skip step |
| `createBranch` | GitHub 403 | "No push access to repository" | Human must fix GitHub permissions |
| `writeFileChange` (single file) | NeuroLink generate error | "File write failed for: {path}" | Retry from `branch_created` |
| `writeFileChange` (single file) | GitHub 409 Conflict | "File SHA conflict on {path}" | Retry from `branch_created` |
| `createPR` | GitHub 422 (PR already exists) | None — treated as success | Skip step |
| `createPR` | GitHub 403 | "No PR creation permission" | Human must fix GitHub permissions |
| `monitorCI` | CI failure | "CI failed: {jobName}" | Human reviews, can retry via `vigilant retry` |
| `monitorCI` | 30-min timeout | "CI monitoring timed out after 30 minutes" | Human reviews CI output |

## Retry Command

```typescript
// Called when user runs: vigilant retry <sessionId>
export async function retryExecutor(
  sessionId: string,
  octokit:   Octokit,
  neurolink: NeuroLink,
): Promise<void> {
  const db      = getStateDb();
  const session = loadSession(db, sessionId);

  if (!session || session.stage !== 'blocked') {
    console.error(`Session ${sessionId} is not in blocked state`);
    return;
  }

  // Reset stage to 'executing' so the daemon picks it up again
  db.prepare(`UPDATE agent_sessions SET stage = 'executing', blocker_reason = NULL WHERE session_id = ?`)
    .run(sessionId);

  // executor_step is NOT reset — executor will skip already-completed steps
  console.log(`Session ${sessionId} queued for retry from step: ${(session as any).executorStep}`);
}
```

## Partial File Write Recovery

If 3 of 5 files were written before a crash:
1. `executor_step = 'branch_created'` (files_written only set after ALL files done)
2. On restart, `writeAllChanges()` is called again for all files
3. Files already written: GitHub returns `200 OK` (update) — idempotent because `PUT /contents/{path}` always succeeds if SHA is current
4. Files not yet written: written fresh

This means **every file is re-generated by NeuroLink** on retry. Acceptable — the generation is deterministic given the same issue and original content.

---

## Non-Recoverable Failures

These are set to `blocked` with a clear reason and should **not** be auto-retried:
- `403 No push access` — permissions issue, not transient
- `403 No PR creation permission` — permissions issue
- CI failure — human must review the CI output, not vigilant's job to guess the fix

The human uses `vigilant session <id>` to view the blocker reason and decide.
